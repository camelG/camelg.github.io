# ワードプレスはもう古い！次に vuepress が来る！そして Why vuepress

長年 wordpress 使っていましたが、最近流行りの vue を学習して、夢中しました。フロント側の開発がこんなに楽しくなるのは初めてで、なんでも vue で書きたくなってしまったことが原因で、初めての vuepress にも挑戦。 　javascript の経験が浅いものの、3 ヶ月くらいで動的ページ、スライドやモーダルコンテンツなどもすぐに作れるようになりました。なんでも欠点から言いたくなるタイプの人ですが、今まで半年くらい使ってデメリットがまだ見つかっていません。なんと魅力的のフレームワークで本当に魅了しました。 　メリットはぐぐったら沢山でますが、自分にとって一番いいのが、なんでも少ないコードで複雑の処理ができることですね。MVVM は最初分からなかったが、練習してすぐに把握しました。 　まあ、本題から離れましたが、why　vuepress というと、まずは今まで使っていた wordpress の欠点から言うと。

# wordpress がだめな点

# スピードが遅い

スピードが遅い

サーバーのスペックもありますが、個人ブログやってるくらいなら、トップページは一秒以内でないと、個人的に気持ちわるい。

# 管理画面がめんどくさい

管理画面がめんどくさい

初心者向けのもあるでしょうか、管理画面が誰でもわかるようにいらない設定、過剰なエディター機能など、機能が沢山ありますが、反応が遅く長く使うとストレスになる

# プラグインが多すぎ

プラグインが多すぎ

魅力のプラグインが沢山あって、本来はメリットになるはずのことですが、いっぱいインストールするとサイトの性能が落ちる

# PWA などの対応が不完全

PWA などの対応が不完全

プラグインをインストールするとすぐに PWA 化できますが、カスタマイズしようとおもったら大変なことで、基本的は PHP で動いているので、pwa 対応の不完全も想定内の話

# vue/vuepress がすごい点

# 静的化で html ファイルを生成できる

静的化で html ファイルを生成できる

html ファイルをコマンドで自動生成できるってかっこいいですね、それだけじゃなくて、js ファイル、画像、css ファイルの圧縮も webpack でやってくれます。より小さい html ファイルを自動に生成はすごい

# vue components 書ける

vue components 書ける

html ファイルですが、vue の components が動く、もちろん ajax 通信でやり取りもできる、MVVM データ連動ももちろん、想像するだけでわくわくする

# 表示が早い

表示が早い

出力するファイルを圧縮しているし、html だけだし、それは表示が早いでしょう

# サーバーいらない

サーバーいらない

PHP でしたらサーバー側用意しないといけない、linux,php,mysql,apache でいわゆる lump 環境が必要ですが、vuepress は必要なし、ローカルで開発して、できたものを generate して deploy するだけ

# markdown ファイルで書く

markdown ファイルで書く

非常に簡単、簡潔、ダイレクトのメモ帳って感じで、メモ帳より機能性が高すぎ

# nodejs 系のサポートがある

nodejs 系のサポートがある

npm コマンドで依存性管理できるので、パッケージを好きにいれて各種機能の管理ができる、これはワードプレスのプラグイン機能みたいな感じ

# pwa は簡単にできる

pwa は簡単にできる

表示するだけなら、すぐにできるし、より高機能の PWA 設定ができるので、これはうれしい

# カスタマイズが自由

カスタマイズが自由

vuepress も vue です。vue が好きな人間にとっては vue ができることはここで全部できるから
